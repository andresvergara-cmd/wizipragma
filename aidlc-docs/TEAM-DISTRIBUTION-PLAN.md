## AIDLC Best Practices for Team

### 1. Trust the Process
- Each stage builds on the previous one
- Don't skip stages (they save time later)
- Artifacts guide implementation

### 2. Review Before Approving
- Read generated artifacts carefully
- Ask questions if unclear
- Suggest changes if needed

### 3. Communicate Changes
- If you change a design decision, update artifacts
- Notify other devs of changes
- Keep aidlc-state.md updated

### 4. Use Agents Effectively
- Agents have context from artifacts
- Ask agents for help with implementation
- Agents can generate boilerplate code

### 5. Test Early and Often
- Don't wait until the end to test
- Test each Lambda function individually
- Test integration points as soon as possible
